Git is the command line version control system. GitHub is an internet hosting service for git repositories. How the Git Credential Managers works: 1.Checks if a personal access token exists in the Windows Credential Manager cache 2.If cached credentials aren’t found, a dialog is shown to get the user’s github credentials. 3.If authentication succeeds, the GCM creates a personal access token with read / write code permissions for the team project. 4.The personal access token is stored in the Windows Credential Manager and used to perform the requests Git command.
